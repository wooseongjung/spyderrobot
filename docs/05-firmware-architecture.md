# 05 — Firmware Architecture (STM32)

Firmware that runs on the custom MCU board. Targets STM32 G4 / F4, written in C, built with STM32CubeIDE / Make.

## Goals

- Deterministic sensor sampling
- Robust framed UART protocol to the Pi
- Clean separation between HAL drivers and application logic
- Watchdog + safety so a hung sensor cannot freeze the system
- Easy to port between STM32 G4 and F4 (HAL-based)

## Layered structure

```
+-------------------------------------------------+
|                  Application                    |
|  - main loop / scheduler                        |
|  - telemetry assembler                          |
|  - safety / watchdog policies                   |
+-------------------------------------------------+
|                Sensor drivers                   |
|  - imu_mpu9250.c   - bme280.c   - ina226.c      |
|  - hcsr04.c        - analog_showcase.c          |
+-------------------------------------------------+
|                 Bus & I/O HAL                   |
|  - i2c_bus.c       - uart_link.c   - adc.c      |
|  - timer_capture.c (HC-SR04)                    |
+-------------------------------------------------+
|         STM32 HAL (CubeMX-generated)            |
+-------------------------------------------------+
```

## Task / scheduler model

- Single super-loop with a 10 ms tick (TIM2 ISR sets a flag).
- Cooperative scheduler dispatches tasks at fixed rates:

| Task | Rate | Notes |
|---|---|---|
| IMU read + filter | 100 Hz | tilt / shock detection |
| Environmental read | 1 Hz | DHT11 / BME280 |
| Ultrasonic ping | 10 Hz | one trigger, capture echo via timer |
| Power monitor | 10 Hz | INA226 shunt + bus voltage |
| Discrete analog showcase ADC | 10 Hz | discrete op-amp current-sense block, parallel to INA226 (ADR-0008) |
| Telemetry frame TX | 10 Hz | bundles latest values |
| Watchdog kick | 10 Hz | only if all critical tasks ran |

> Could move to FreeRTOS later, but a super-loop is enough for v1 and easier to debug.

## UART telemetry frame format (draft)

Little-endian, fixed prefix, length, payload, CRC16.

```
+------+------+--------+----------+--------+
| 0xAA | 0x55 | LEN(1) | PAYLOAD  | CRC16  |
+------+------+--------+----------+--------+
        SYNC    LEN     N bytes    2 bytes
```

Payload v1 (TODO: lock down once sensors are wired):

| Offset | Field | Type | Notes |
|---|---|---|---|
| 0 | seq | u16 | rolling counter |
| 2 | t_ms | u32 | milliseconds since boot |
| 6 | temp_c | i16 | × 100 |
| 8 | rh | u16 | × 100 |
| 10 | dist_mm | u16 | HC-SR04 |
| 12 | analog_showcase_ua | i32 | discrete current-sense block (ADR-0008) |
| 16 | accel[3] | i16 | mg |
| 22 | gyro[3] | i16 | dps × 10 |
| 28 | bus_mv | u16 | INA226 |
| 30 | shunt_ua | i32 | INA226 |
| 34 | flags | u8 | bit0 = fall, bit1 = lowbat, etc. |

CRC16-CCITT over LEN..PAYLOAD.

## Command protocol (Pi → MCU)

| Cmd | ID | Payload | Effect |
|---|---|---|---|
| PING | 0x01 | — | MCU replies with PONG + uptime |
| GET_INFO | 0x02 | — | firmware version, MCU ID |
| SET_RATE | 0x10 | task_id, hz | change task rate |
| RESET | 0xF0 | magic | software reset (with NVIC_SystemReset) |

## Safety / watchdog

- IWDG enabled at boot (~ 1 s timeout).
- Each critical task must run within its window or the watchdog kick is suppressed.
- On reset cause = IWDG, MCU latches an error code in backup register and TX a `BOOT_AFTER_WDG` event on next link.

## Build / toolchain

- STM32CubeIDE (initial) → optionally migrate to PlatformIO + arm-none-eabi-gcc
- HAL generated by CubeMX
- Source goes in `firmware/mcu/`
- Tests (host-side, mock HAL) in `firmware/tests/`

## Style

- C11, `-Wall -Wextra -Wpedantic`
- No dynamic allocation in steady state
- All driver init functions return a status enum
